## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel env store → app runtime | Segredos (JWT, service_role) injetados no runtime server-only |
| Supabase Auth → email de convite | Redirect URL controla para onde o link do convite leva |
| Browser → Edge Function | CORS via APP_URL controla origens permitidas |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information disclosure | SUPABASE_ROLE_KEY / SUPABASE_JWT | mitigate | Task 1 confirma que nunca são prefixados NEXT_PUBLIC_ — service_role bypassa RLS e não pode vazar ao client |
| T-06-02 | Spoofing | Supabase Auth Redirect URL | mitigate | Task 2/3 restringem Redirect URL ao domínio de prod exato (romma-alpha.vercel.app/**), evitando open-redirect no link de convite |
| T-06-03 | Information disclosure | CORS da Edge Function | accept | APP_URL define origem única de prod; localhost permanece para dev — risco baixo, sem dados sensíveis na resposta |
</threat_model>
