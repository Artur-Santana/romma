# Milestone v1.0: TCC Finalization

**Status:** ✅ SHIPPED 2026-06-03
**Phases:** 1-7
**Total Plans:** 29

## Overview

Romma v1.0 entrega o sistema completo de gerenciamento de aluguéis corporativos como TCC. Ciclo completo implementado: listagem pública de Unidades → Portal do Locatário → Dashboard do Proprietário com valores financeiros reais → Deploy estável em produção → Suite de testes E2E → Roteiro de banca. Sistema demonstrável ao vivo em romma-alpha.vercel.app.

## Milestone Summary

**Key Decisions:**
- Design system Obsidian Blueprint (roxo #370085 / dourado #C5A059) com Manrope/Noto Sans adotado consistentemente
- proxy.js em vez de middleware.js (Next.js 16 breaking change)
- 5 clientes Supabase separados por contexto (evita service role no browser)
- Realtime com limitação conhecida (disponivel→alugada não propaga via RLS — card some no reload)
- /auth/confirm como Route Handler em vez de middleware (Next.js 16 App Router pattern)
- seed-prod-demo.mjs idempotente para demo da banca sem poluir DB

**Issues Resolved:**
- IDOR crítico em cancelarContrato/encerrarContrato (unidade_id não validado)
- Mass assignment em editarLocatario (campos raw sem allowlist)
- erroMessage vs errorMessage (typo no contratos.js)
- Fluxo de convite quebrado em produção (/auth/confirm faltando)
- UTC-3 offset bug em alerta contratos vencendo (getTodayLocal)

**Issues Deferred:**
- UAT manual Phase 04/05 (validações visuais de UI)
- Verification gaps human_needed em fases 01/04/05
- VIS-04: Landing Page com Obsidian Blueprint completo (pós-banca)
- DEMO-02/DEMO-03 (dados de demo realistas e validação ponta a ponta)

**Technical Debt Incurred:**
- criarContrato + gerarParcelas não atômicos (sem rollback se Edge Function falhar)
- Realtime UPDATE (disponivel→alugada) não propaga — limitação conhecida do RLS
- 8 warnings no-img-element na landing page (/) — adiado por não impactar banca
