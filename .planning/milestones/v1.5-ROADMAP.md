## Overview

O design system v1.5 ("r-*" classes + tokens de densidade) está 100% aplicado em todas as 9 fases.

## Milestone Summary

### Key Decisions

- **Tokens CSS primeiro** — `--rt-*` / `--rd-*` em globals.css como raiz de dependência; zero regressão em telas não refatoradas ainda
- **FauxQR vs imagem estática** — QR 25×25 determinístico (LCG seeded do código PIX) — sem asset externo, visual consistente com o valor
- **jsPDF import dinâmico** — `await import('jspdf')` dentro do click handler — SSR-safe, lazy-load sem overhead em build
- **Guard 3-hop Locatário** — parcela → contrato → locatario → usuario_id, retorna 404 cross-tenant (espelha padrão 4-hop Proprietário)
- **Renovar contrato via append** — novas parcelas appended, pagas preservadas — sem re-chamar Edge Function gerar-parcelas
- **"Falar com Proprietário" vs "Simular aluguel"** — pivot correto: visitante não aluga, o Proprietário fecha — unidade permanece na lista
- **Signed URLs para fotos** — Storage privado + signed URL por cadeia de propriedade; next/image com remotePatterns para o bucket
- **design/portal.jsx Variant B** — referência canônica para fidelidade pixel-level do portal do Locatário

### Issues Resolved

- Scroll mobile estoura em flex containers — corrigido com `min-height: 0` em toda a cadeia
- Modais não centralizavam no mobile — corrigido com `position: fixed; inset: 0`
- Animações deixavam `opacity: 0` como fill final (invisível em projetor/print) — corrigido com `animation-fill-mode: none`
- Realtime UPDATE disponivel→alugada ignorado por RLS — limitação documentada, comportamento aceitável para banca
