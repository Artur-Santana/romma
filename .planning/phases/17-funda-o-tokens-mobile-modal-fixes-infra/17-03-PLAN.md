## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client to storage.objects | A logged-in proprietario uploads/reads/deletes files; must only reach files under unidades they own |
| anon to storage.objects | Public visitors must NOT read private bucket objects directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Information Disclosure | storage.objects SELECT (IDOR on another proprietario's photo) | mitigate | SELECT policy requires `bucket_id='unidades-fotos' AND public.storage_unidade_owned_by_auth(storage.objects.name)`; bucket is PRIVATE so no anon direct URL access |
| T-17-02 | Tampering | storage.objects INSERT (cross-proprietario upload) | mitigate | INSERT WITH CHECK calls the same ownership-chain function before allowing the write |
| T-17-03 | Tampering | path injection in storage path | mitigate | path convention `{unidade_id}/{filename}`; `storage.foldername(name)[1]` extracts segment; uuid cast in EXCEPTION-guarded function returns FALSE on malformed input |
| T-17-04 | Elevation of Privilege | SECURITY DEFINER search-path injection | mitigate | `SET search_path = public, storage` pinned on the function |
| T-17-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (RESEARCH Package Legitimacy Audit: none required) |
</threat_model>

Then REVOKE EXECUTE from anon and GRANT EXECUTE to authenticated. (5) Three RLS policies on storage.objects named unidades_fotos_insert_proprietario (FOR INSERT TO authenticated WITH CHECK), unidades_fotos_select_proprietario (FOR SELECT USING), unidades_fotos_delete_proprietario (FOR DELETE USING), each gated by `bucket_id = 'unidades-fotos' AND public.storage_unidade_owned_by_auth(storage.objects.name)` — qualify storage.objects.name to avoid Pitfall 4 ambiguity. Precede each CREATE POLICY with DROP POLICY IF EXISTS of the same name ON storage.objects for idempotent re-runs. (6) GRANT ALL ON public.proprietarios and public.unidades TO service_role, authenticated. Do NOT write any Server Action / app code here (Phase 19).
