## Interaction Contract (mobile-specific)

This section is critical for Phase 13. These are the prescriptive rules the executor must follow.

### Breakpoints

| Breakpoint | Value | Behaviour |
|-----------|-------|-----------|
| Mobile min viewport | 375px | Target — iPhone SE / most Android |
| CSS media query (shell switch) | `max-width: 768px` | Already in globals.css: `.romma-sidebar-wrapper { display: none }`, `.romma-mobile-only { display: flex; flex-direction: column; height: 100vh }` |
| Tailwind `sm:` | 640px | Use for layout transitions (padding, typography) |

### DashboardShell

| Condition | Layout |
|-----------|--------|
| Desktop (> 768px) | `TopStrip` + sidebar (`OwnerSidebar`) + `<main>` — identical to current `layout.js` |
| Mobile (≤ 768px) | `MobileTopBar` + `<main style="flex:1;overflow:auto">` + `MobileBottomNav` — full `height: 100vh`, no sidebar |

**Rule:** Sidebar must not render, overflow, or overlap at 375px. The `.romma-sidebar-wrapper { display: none }` CSS guarantees this only if `DashboardShell` preserves the classname on the sidebar wrapper.

**Detection approach:** CSS-first using existing `.romma-desktop-only` / `.romma-mobile-only` utility classes (render both, CSS hides the inactive one). Hook `useMediaQuery(768)` is acceptable if CSS approach produces hydration issues — implementor decides.

### MobileTopBar — title map (D-03)

| Route | Title | onBack |
|-------|-------|--------|
| `/dashboard` | "Dashboard" | omitted |
| `/dashboard/unidades` | "Unidades" | omitted |
| `/dashboard/contratos` | "Contratos" | omitted |
| `/dashboard/locatarios` | "Locatários" | omitted |
| `/dashboard/contratos/[id]` | "Parcelas" | `router.back()` |

No subtitle on any route (omitted per D-03).

### MobileBottomNav — items (D-02)

```js
[
  { href: "/dashboard",           label: "Início",      code: "OVW" },
  { href: "/dashboard/unidades",  label: "Unidades",    code: "UNI" },
  { href: "/dashboard/contratos", label: "Contratos",   code: "CTR" },
  { href: "/dashboard/locatarios",label: "Locatários",  code: "LOC" },
]
```

Active detection: `pathname === item.href || pathname.startsWith(item.href + "/")` — already in MobileNav.js, use as-is.

### Overflow & Scrollable Containers

| Component | Fix |
|-----------|-----|
| `Contratos.js` — table rows | Wrap table in `overflow-x: auto` container |
| `Parcelas.js` — table rows | Wrap table in `overflow-x: auto` container |
| `LocatariosDesktop.js` — table | Wrap table in `overflow-x: auto` container |
| `Unidades.js` — table/list | Wrap table in `overflow-x: auto` if it has a horizontal table |
| `ParcelsTable.js` (portal) | Wrap table in `overflow-x: auto` container |

**Rule:** No horizontal page-level scroll at 375px viewport. Scroll is permitted only inside wrapped table containers.

### Responsive Padding — Portal (D-05)

Apply these Tailwind class replacements to `PortalDashboard.js`:

| Current class | Replace with |
|--------------|-------------|
| `px-12` | `px-4 sm:px-12` |
| `pt-12` | `pt-6 sm:pt-12` |
| `text-[48px]` | `text-[28px] sm:text-[48px]` |

Check `ContratoCard.js` and `ParcelsTable.js` for hard-coded `px-12` equivalents and apply same pattern.
