## Interaction Contract

### Route Architecture

| Route | Auth | Description |
|-------|------|-------------|
| /login | Public (redirect if authed) | Shared login page — already exists |
| /portal | Protected (layout.js guard) | Portal shell with TopStrip |
| /portal/dashboard | Protected | Locatário home: contrato + parcelas |

**Login redirect decision (conflict resolution):**
The existing /login page hardcodes `router.push("/dashboard")` post-auth. Phase 2 must change this to role-based routing:
- If `is_proprietario()` RPC returns true → redirect to `/dashboard`
- Otherwise → redirect to `/portal/dashboard`
This change applies to the existing `/login/page.js`. The UI spec is the same — only the post-auth redirect logic changes.

### States to Implement

| Screen | State | UI Behavior |
|--------|-------|-------------|
| /portal/dashboard | Loading | Skeleton placeholders for contrato card + parcelas table |
| /portal/dashboard | Contrato ativo encontrado | Show ContratoCard + ParcelsTable |
| /portal/dashboard | Sem contrato ativo | Empty state (see Copywriting) |
| /portal/dashboard | Erro de fetch | Error banner (see Copywriting) |
| ParcelsTable | Default | Shows rows: paga, pendente, vencida — ordered by data_vencimento DESC |
| ParcelsTable | Empty (0 parcelas visíveis) | Inline message (see Copywriting) |

### Mobile vs Desktop

Portal is a read-only view — no sidebar needed. Full-width single-column layout on mobile. Centered constrained container (max-w-2xl) on desktop, consistent with the data-dense but minimal approach of the login page.

No mobile/desktop feature split for Phase 2 — content is identical, layout is responsive.
