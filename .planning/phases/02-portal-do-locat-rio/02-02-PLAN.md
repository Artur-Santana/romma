| Boundary | Description |
|----------|-------------|
| browser → /login | Credenciais user-provided; auth via Supabase signInWithPassword (HTTPS) |
| proxy.js → Supabase RPC | createServerClient lê cookies httpOnly via @supabase/ssr |
| /portal/** ← browser | Guard duplo (proxy.js + layout.js defense-in-depth) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Elevation of Privilege | proxy.js /portal block | mitigate | RPC `is_proprietario()` chamado server-side com cookie autenticado; redireciona Proprietário para /dashboard antes do render |
| T-02-05 | Spoofing | /portal/** sem cookie | mitigate | proxy.js redireciona para /login antes de renderizar layout; layout.js valida `getUser()` como defense-in-depth (D-04) |
| T-02-06 | Information Disclosure | login.handleSubmit RPC | accept | RPC `is_proprietario()` retorna apenas boolean — não vaza identidade de outros usuários |
| T-02-07 | Tampering | router.push manipulado client-side | mitigate | Mesmo que o cliente force `router.push('/dashboard')` manualmente, proxy.js bloqueará Locatário em /dashboard via guard existente (Phase 1) |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | Nenhuma instalação de pacote nova (RESEARCH § Package Legitimacy Audit) — não aplicável |
