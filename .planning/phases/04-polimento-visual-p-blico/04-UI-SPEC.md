## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Compact element spacing, inline gaps |
| md | 16px | Default element spacing, modal inner padding |
| lg | 24px | Section padding, card vertical rhythm |
| xl | 32px | Layout gaps, page-level padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (public page hero) |

Exceptions:
- **20px (`px-5` / `py-5`)** — established card padding from Fase 2 canonical refs (`ContratoCard.js`, `LocatariosDesktop.js`). Preserve in `UnidadePublicaCard.js` and `UnidadeDetailSheet.js`.
- **14px (`py-[14px]`)** — established bottom sheet button padding from current `unidades/page.js`. Preserve in `UnidadeDetailSheet.js`.
- **44px touch target minimum** — interactive elements in the public bottom sheet (`UnidadeDetailSheet`) must meet 44px minimum touch height on mobile.

## Typography

| Role | Size | Weight | Line Height | Class Pattern |
|------|------|--------|-------------|---------------|
| Label / Eyebrow (mono, uppercase, tracked) | 9–11px | 700 | 1.4 | `font-mono text-[11px] tracking-[1px] uppercase` |
| Body | 13px | 400 | 1.5 | `font-body text-[13px]` |
| Heading (card titles, section heads) | 22px | 700 | 1.2 | `font-body font-bold text-[22px] tracking-[-0.8px]` |
| Display (page title, detail sheet unit name) | 32px | 700 | 1.1 | `font-body font-bold text-[32px] tracking-[-1.6px]` |

Notes:
- Two weights only: 400 (regular) and 700 (bold). The `font-medium` (500) used in a few places in `LocatariosDesktop.js` is a grandfathered exception — do not introduce new 500-weight usages in Phase 4 components.
- Label row covers the 9–11px mono/eyebrow range. Use 9px for ref tags (UN-XXXXXX), 10px for table headers, 11px for standard eyebrows. All uppercase with tracking.
- The 56px hero display (login page) is landing-page-only and out of scope for this phase.

Sources: `src/components/features/portal/ContratoCard.js`, `src/components/features/LocatariosDesktop.js`, RESEARCH.md Padrão 4.

| Dominant (60%) | `--background` / `bg-background` | #1F1F1F | Page background, document surface |

1. Primary CTA button background ("Tenho interesse →" in `UnidadeDetailSheet`)

4. Focus ring / `--ring` token on shadcn Input and Select

Gold (`--color-highlight`) reserved for:
1. Optional visual accent on unit reference tags in `UnidadePublicaCard` (e.g. area badge, ref code) — one element per card maximum.

## Interaction Patterns

### Bottom Sheet (UnidadeDetailSheet)

- Trigger: click on `UnidadePublicaCard`
- Overlay: `fixed inset-0 z-50` (full viewport — intentional upgrade from current `position: absolute` scoped to container)
- Close: click outside overlay OR "Fechar" button
- Animation: slide up from bottom on mobile — `translateY` transition using `--ease-crisp` (300ms)
- `z-index` safety: verify no conflicts with any fixed header on the public page

### Inline Edit (UnidadeCard — dashboard)

- Mode toggle: click "Editar" → card switches to edit mode (controlled by `editandoId` state in parent `Unidades.js`)
- Fields: shadcn Input for text fields, Select for `status` ENUM
- Save: "Salvar" calls parent `handleSalvarUnidade`; returns to read mode on success
- Cancel: "Cancelar" clears `editandoId` without saving
- Error: displayed below the form, `text-danger-fg text-[11px] font-mono`

### Edit Modal (LocatariosDesktop — D-09)

- Pattern: mirror existing invite modal (`showInviteForm`) — same overlay/modal structure
- State: `editandoId` (useState, separate from form) + `formEdit` object (5 fields: `nome_razao_social`, `tipo`, `documento`, `email`, `telefone`)
- Trigger: "Editar" button in each locatário row, consistent with existing action buttons (VER →, Pendente, etc.)
- Save: calls `editarLocatario` Server Action → on 200: close modal + reload locatários
- Error: `erroMessage` from action displayed in modal, same pattern as invite form

## Design Constraints

| Constraint | Source | Rule |
|-----------|--------|------|
| No inline styles (except button reset) | CLAUDE.md + CONTEXT.md D-04 | All layout via Tailwind v4 utility classes. Exception: `style={{ all: 'unset', cursor: 'pointer', ... }}` for button reset only. |
| Sharp corners | `globals.css` `--radius: 0` | All shadcn components inherit `rounded-none`. Do not override with rounded classes. |
| Dark-only design | `globals.css` — no light mode tokens | Do not add light mode variants. All tokens assume dark surface. |
| Obsidian Blueprint palette | CONTEXT.md D-06 | Use only `--indigo`, `--color-highlight`, `--fg-*`, `--surface*`, `--border-*` tokens. No raw hex values in components. |
