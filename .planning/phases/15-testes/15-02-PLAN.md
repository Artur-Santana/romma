<objective>
[BLOCKING PREREQUISITE for D-08.] Close the real multi-tenant IDOR in `unidades.js` and `contratos.js`. Today `authGuard()` returns `{}` (not `{ user }`), and the mutation Actions filter only `.eq('id', id)`. Because `supabaseAdmin` uses the service-role key (bypasses RLS), any authenticated Proprietário can edit/delete a unidade or cancel/encerrar a contrato belonging to another Proprietário. This is the gap RESEARCH flagged as Critical Finding and the reason D-08 assertions are currently impossible for these files. `unidades`/`contratos` have NO `proprietario_id` column — tenancy flows via `edificio_id → edificios.proprietario_id`. The fix: make `authGuard()` return `{ user }`, then add an ownership pre-check before each mutation.

This is a production code change but is in scope and REQUIRED to fulfill D-08 (per planning context). MT-02 in Phase 11 closed IDOR for edificios/locatarios but missed these four Actions.

Purpose: Close a live cross-tenant write vulnerability and unblock the D-08 assertions in plan 15-04.
Output: Ownership-scoped mutations in both Action files.
</objective>
