| Hover | same as inactive + `background: var(--surface-hi)` | — | — | — |

Counter in tab = count of `available` for that edificio (excludes `removedIds`). Updates live on simular.

### Sort Pills (PUB-02)

| State | Border | Background | Text color |
|-------|--------|-----------|------------|
| Inactive | `1px solid var(--border-3)` | `transparent` | `var(--fg-4)` |
| Active | `1px solid var(--indigo)` | `oklch(0.339 0.179 301.68 / 0.18)` | `var(--fg-1)` |

Sort IDs: `rel` (default, no-op), `valor_asc`, `valor_desc`, `area_desc`. Labels: "Relevância", "Menor valor", "Maior valor", "Maior área". Null-value guard required (see RESEARCH.md Pitfall 3).

### Card (PUB-03)

- Entire card is clickable → opens sheet
- `cursor: pointer`
- Hover: `background: var(--surface-hi)` via `.r-rowlink` class
- Removing: `transition-opacity duration-700` → `opacity-0` → removed from DOM after 700ms

### Sheet Open / Close (PUB-04)

- Opens on card click: `setSelected(unidade)`
- Sheet entrance: `animation: rSheetUp 320ms var(--ease-crisp) both` on panel element
- Close: backdrop click or "✕" button → `setSelected(null)`
- Backdrop: `oklch(0 0 0 / 0.7)` fixed inset-0 z-60
- Sheet panel: `fixed bottom-0 w-full maxWidth:520 bg-surface borderTop-indigo maxH-92% overflowY-auto`
- Sheet does NOT use `.romma-modal-backdrop` (different alignment — aligns to bottom, not center)

### Simular Aluguel (PUB-05)

**Sequence:**
1. User clicks "[>] Simular Aluguel"
2. `simulating` state → `true` → button shows "[···] Processando", disabled
3. After ~800ms: `simulating → false`, sheet closes (`setSelected(null)`), `removingId = uid`
4. Card opacity transitions to 0 over 700ms
5. After 700ms: `removedIds.add(uid)`, `removingId = null`, card removed from render
6. Tab counters update (derived from `available` which excludes `removedIds`)

**`simulating` state location:** `UnidadesPublicas.js` (parent), not encapsulated in sheet. Passed as prop to sheet. Rationale: unified control with `simularAluguel()` flow (RESEARCH.md risk table).

## Foto URL Resolution Strategy

**All URL resolution happens server-side (in RPC), not client-side.**

The RPC `get_unidades_disponiveis()` must be updated (Plano 24-01) to return `foto_signed_url text` alongside `foto_url text`. The SECURITY DEFINER context generates the signed URL via `storage.sign()`.

**Client-side resolution (sync, no async calls):**

```js
// In useEffect after load:
const fotoSrcs = Object.fromEntries(
  unidades.map(u => [
    u.id,
    u.foto_signed_url ?? u.foto_url ?? '/Detalhe_Arquitetonico.png'
  ])
)
setFotoSrcs(fotoSrcs)
```

`resolveFotoUrl` is NOT async. No `createSignedUrl` calls from the anon client — the bucket is private and anon has no SELECT policy on `storage.objects`.

- Card image: `alt=""` (decorative — unit name is in adjacent text)
- Sheet image: `alt=""` (decorative — unit name is in adjacent heading)
- CTA button: disabled during `simulating` state
- Touch targets: CTA button `minHeight: 44px` (iOS HIG minimum)
- Close button: 30×30 visible target (below 44px minimum — acceptable for supplementary action; backdrop click is primary close path)
- Tab overflow: `overflowX: auto` with `scrollbar-width: none` (`.r-noscroll`) — keyboard-accessible via Tab key
