**Accent (`--ds-primary` / `--indigo`) reserved for:**
- Eyebrow labels marked `eyebrow--indigo`
- Active nav item indicator
- Primary action button (shadcn Button with `variant="default"`)
- Focus ring (`--ring`)
- Sidebar primary item highlight

**Accent is NOT used for:** body text, borders, card backgrounds, secondary buttons, logout button.

The logout button uses foreground text color (`--fg-3` or `--muted-foreground`) — it is a low-emphasis action, not a primary CTA.

## Interaction Contract

### Logout Button (D-06 — the only new interactive element)

| Property | Spec |
|----------|------|
| Placement | Inline in `PortalDashboard.js` — top-right of the portal header block, aligned with the eyebrow row |
| Component | `LogoutButton.js` — new Client Component with `'use client'` directive |
| Label | `Sair` (text only — no icon required; icon optional: `ri-logout-box-line` from remixicon) |
| Visual style | Low emphasis — `style={{ all: "unset", cursor: "pointer" }}` pattern + `font-mono text-[10px] text-fg-4 tracking-[1px] uppercase hover:text-fg-2 transition-colors` to match TopStrip monospace aesthetic |
| Interaction | Click → `await supabase.auth.signOut()` → `router.push('/login')` |
| Loading state | Button text changes to `Saindo...` during async signOut (single useState boolean) |
| Error state | On signOut failure: show inline text `Erro ao sair. Recarregue a página.` in `--danger-fg` color below the button — no toast/dialog needed |
| Confirmation dialog | None — logout is not destructive of user data |
| Import | Use `supabase = createClient()` already instantiated at top of PortalDashboard.js — pass it as prop to LogoutButton or re-instantiate inside LogoutButton |
| Router | `useRouter` from `next/navigation` |

**Rationale for PortalDashboard placement (not TopStrip):** TopStrip is a pure-display monospace status bar with semantic meaning (system node timestamp). Adding an interactive auth action breaks its single responsibility. PortalDashboard is already a Client Component with the supabase instance — zero new dependencies required.
