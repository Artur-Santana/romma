## Surface Specifications

### Surface 1: VencimentoDestaque (novo componente)

**Purpose:** Highlight card for the next payable installment. Appears between the page header and `ContratoCard`.

**Layout:**
```
[PRÓXIMO VENCIMENTO]   ← eyebrow--indigo, 10px uppercase
──────────────────────── border-left accent (4px solid --highlight)
R$ X.XXX,XX             ← font-display bold 32px fg-1
Parcela 03/12 · 14 dias restantes   ← font-mono 11px fg-3 (dias em --warning se ≤7d, --danger-fg se <0)
[>] PAGAR AGORA         ← button, indigo bg, bracket notation
```

**States:**
- Normal (≥ 8 dias): dias restantes in `text-fg-3`
- Urgente (≤ 7 dias): dias restantes in `text-warning`
- Vencida (dias < 0): dias restantes shows "X dias em atraso" in `text-danger-fg`; "PAGAR AGORA" still shows (pendente/vencida both payable)
- Sem parcela pagável: component is hidden (no `futura`-only state shown — only pendente/vencida trigger this card)

**Tailwind classes:**
- Container: `mt-8 border-l-4 border-l-highlight bg-surface p-6 flex flex-col gap-3`
- Eyebrow: `eyebrow eyebrow--indigo`
- BRL value: `font-display font-bold text-[32px] leading-none tracking-[-1.6px] text-fg-1`
- Subtitle: `font-mono text-[11px] text-fg-3` (dias: conditional `text-warning` or `text-danger-fg`)
- CTA button: `bg-indigo text-fg-1 font-mono font-bold text-[11px] tracking-[1.5px] uppercase px-5 py-3 cursor-pointer hover:opacity-90 transition-opacity w-full sm:w-auto mt-2`

**Accessibility:** `aria-label="Próximo vencimento: R$ X, parcela 3 de 12, vencimento em 14 dias"` on container. Button has visible text — no icon-only.

### Surface 2: ContratoCard — progresso row (extensão)

**Adição abaixo do grid existente:**
```
[barra de progresso — track bg-border-3, fill bg-indigo scaleX animated]
PAGAS: 03  /  TOTAL: 12  ·  25% ADIMPLENTE    ← font-mono 11px fg-4 / highlight
```

**Progress bar spec:**
- Height: 6px
- Container: `w-full bg-border-3 mt-4`

**Stats row:** `flex gap-4 mt-2`
- "PAGAS: N": `font-mono text-[11px] text-fg-4`
- " / TOTAL: N": `font-mono text-[11px] text-fg-4`
- "· Xadimplente": `font-mono text-[11px] text-highlight` (gold) when < 100%, `text-success` when 100%

### Surface 4: PixModal (novo componente)

**Trigger:** "PAGAR AGORA" in `VencimentoDestaque` OR `[>] PAGAR` in `ParcelsTable` row.

**Structure:**
```
.romma-modal-backdrop  (position:fixed; inset:0; z-50; bg-black/70; flex center)
  └── modal body  (bg-surface border border-border-3 w-full max-w-md p-7)
        ├── [header]
        │     eyebrow: PAGAMENTO VIA PIX
        │     title:   R$ X.XXX,XX  (32px display bold)
        │     sub:     Parcela 03/12 · Vencimento: DD/MM/AAAA
        ├── [QR section]
        │     <img src="/pix-qr.png" alt="QR Code PIX estático" />
        │     width: 200px; display: block; margin: 0 auto
        ├── [código copia-e-cola]
        │     label: CÓDIGO PIX (eyebrow fg-4)
        │     code box: bg-background border border-border-3 px-4 py-3
        │                font-mono text-[11px] text-fg-3 break-all select-all
        │     button: [>] COPIAR CÓDIGO  →  [✓] COPIADO  (2s reset)
        ├── [nota de demo]  ← OBRIGATÓRIO por PORT-05
        │     "Este é um ambiente de demonstração. O pagamento real não é
        │      processado. Clique em confirmar para registrar o pagamento."
        │     font-mono text-[11px] text-fg-4 italic border-l-2 border-warning pl-3
        ├── [actions row]
        │     [CANCELAR]     ghost btn — font-mono text-[11px] text-fg-4 border border-border-3 px-5 py-3
        │     [✓] CONFIRMAR  filled btn — bg-indigo font-mono font-bold text-[11px] tracking-[1.5px] uppercase px-5 py-3
        │     loading state: "[···] CONFIRMANDO"
        │     success: briefly shows "[OK] 200" then closes modal + re-fetch parcelas
```

**State machine:**
- `idle` → "CONFIRMAR" clickable
- `loading` → button shows "[···] CONFIRMANDO", both buttons disabled
- `erro` → inline error below actions: `font-mono text-[11px] text-danger-fg`; "CANCELAR" re-enabled
- `sucesso` → close modal, re-fetch `todasParcelas`, show sonner toast "Pagamento registrado"

**Focus trap:** Modal must trap focus (Tab cycles within modal). Close on Escape key.

**Mobile (375px):** Modal is full-width with `mx-4` constraint. QR image 160px on mobile. Actions stack vertically (`flex-col gap-3`).

**Animation:** Modal entrance — `rSheetUp` from `@keyframes rSheetUp` (existing, slides up 320ms). `@media (prefers-reduced-motion: reduce)` — no animation, modal appears immediately.

### Surface 5: Recibo PDF (jsPDF layout)

Generated in-browser via `handleBaixarRecibo`. Not a screen — a document layout.

**Document spec:**
- Page size: A4 portrait
- Margins: 20mm left/right, 20mm top/bottom
- Font: helvetica (jsPDF built-in — no font embedding required for TCC)

**Content layout (y positions in mm):**

```
y=15  ROMMA — COMPROVANTE DE PAGAMENTO      [helvetica bold 14pt, centered]
y=22  ─────────────────────────────────     [doc.line(20, 22, 190, 22)]

y=32  DADOS DA PARCELA                      [helvetica bold 10pt, uppercase]
y=40  Locatário:    {nome_razao_social}      [helvetica normal 10pt]
y=48  Unidade:      {unidade.nome}
y=56  Parcela:      {numero} de {total}
y=64  Valor mensal: R$ {valor_mensal fmtBRL}

y=76  DATAS                                 [helvetica bold 10pt, uppercase]
y=84  Fechamento:   {data_fechamento fmtData}
y=92  Vencimento:   {data_vencimento fmtData}
y=100 Pagamento:    {data_pagamento fmtData}

y=112 PAGAMENTO                             [helvetica bold 10pt, uppercase]
y=120 Forma:              PIX
y=128 Código de auth.:    {gerarCodigoAuth(id, data_pagamento)}

y=145 ─────────────────────────────────     [doc.line]
y=152 Este documento é gerado automaticamente pelo sistema Romma.
      Código de autenticação identifica unicamente este comprovante.
      [helvetica normal 8pt, text-gray / color [120,120,120]]
```

**File name:** `recibo-parcela-{String(numero).padStart(2,'0')}.pdf`

**gerarCodigoAuth function:** Deterministic hash using btoa approach (D-05 "alternativa mais robusta"):
`btoa(parcelaId.slice(0,8) + (dataPagamento ?? '')).replace(/[^A-Z0-9]/gi,'').toUpperCase().slice(0,8)`
Example output: `"A3F2B1C9"`. Same parcela+date always produces the same code.

## Interaction Contracts

### IC-01: "Pagar Agora" flow

1. User clicks `[>] PAGAR AGORA` (VencimentoDestaque) or `[>] PAGAR` (ParcelsTable row)
2. `PixModal` opens with parcela data injected: valor, número, data_vencimento
3. User optionally clicks `[>] COPIAR CÓDIGO` — clipboard write, button changes to `[✓] COPIADO` for 2s, then resets
4. User clicks `[✓] CONFIRMAR`
5. Loading state: `[···] CONFIRMANDO`; both buttons disabled
6. Server Action `confirmarPagamentoLocatario(parcela.id)` called
7. On `{ status: 200 }`: modal closes, re-fetch `todasParcelas`, sonner toast "Pagamento registrado"
8. On error: inline error message, buttons re-enabled, modal stays open

### IC-02: "Baixar comprovante" flow

1. User clicks `[↓] RECIBO` (ParcelsTable row for `paga` parcelas)
2. `handleBaixarRecibo(parcela, locatario, contrato, totalParcelas)` called
3. Dynamic import: `const mod = await import('jspdf'); const JsPDF = mod.jsPDF ?? mod.default?.jsPDF ?? mod.default`
4. PDF built and `doc.save(...)` triggers browser download
5. On error: `setErro('Não foi possível gerar o comprovante. Tente novamente.')` shown inline below the table action

### IC-03: Clipboard copy

- Uses `navigator.clipboard.writeText(PIX_CODE_CONST)` in `try/catch`
- On success: button text → `[✓] COPIADO`, reset after 2000ms via `setTimeout`
- On failure (non-HTTPS or browser block): button text → `[!] ERRO AO COPIAR` for 2s, then reset — no crash

### IC-04: Modal close

- Escape key closes modal (keydown listener on mount, removed on unmount)
- "CANCELAR" button closes modal
- Click on `.romma-modal-backdrop` does NOT close (prevents accidental dismiss mid-flow)
- On close: reset modal state (`loading: false`, `erro: null`, `copiado: false`)
