<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user file input → browser | untrusted file (type/size/name) selected via input or drag-drop |
| browser → Supabase Storage (anon) | direct upload to private bucket `unidades-fotos`; RLS enforces ownership |
| browser → Server Action | only the path string + form fields cross; binary never reaches the server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-07 | Tampering | file upload (arbitrary exe/script) | mitigate | client validates `file.type.startsWith('image/')` + `size < 2MB`; explicit `contentType` on upload (V5) |
| T-19-08 | Tampering | path traversal via filename | mitigate | path uses `crypto.randomUUID()` + extracted extension; `file.name` never used in the Storage path (V5) |
| T-19-09 | Elevation of Privilege | IDOR on upload (owner A → owner B's unidade) | mitigate | path is `{unidade_id}/...`; RLS `storage_unidade_owned_by_auth` validates unidade→edificio→proprietario_id; create returns id only after authGuard insert (V4) |
| T-19-10 | Information Disclosure | signed URL TTL leak | accept | `expiresIn: 3600`, bucket stays private; not persisted across sessions (per RESEARCH Pitfall 4) — accepted for TCC |
| T-19-SC | Tampering | npm/pip/cargo installs | mitigate | no new packages (RESEARCH Package Legitimacy Audit: none) — no install task |
</threat_model>
