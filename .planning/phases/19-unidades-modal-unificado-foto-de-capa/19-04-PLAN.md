## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Supabase Storage (anon) | signed-URL generation + best-effort delete cleanup on the private bucket |
| browser → Server Action | delete passes only the unidade id; ownership re-checked server-side |
| owner dashboard → rendered data | foto_url paths rendered as signed URLs; valor_visivel respected for display |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-11 | Elevation of Privilege | IDOR on delete | mitigate | deletarUnidade (Plan 01) verifies `edificio.proprietario_id === user.id` before any delete; client cleanup is best-effort only (V4) |
| T-19-12 | Information Disclosure | signed URL on cards | accept | `expiresIn: 3600`, re-generated on render, never persisted; private bucket retained (RESEARCH Pitfall 4) |
| T-19-13 | Tampering | client-side Storage cleanup | mitigate | guarded by `!foto_url.startsWith('/')`; failures swallowed (best-effort, D-11); server-side cleanup in Plan 01 is the canonical path |
| T-19-SC | Tampering | npm/pip/cargo installs | mitigate | no new packages (RESEARCH Package Legitimacy Audit: none) — no install task |
