## Spacing Scale

> **Inherited system.** The existing public pages use non-strict spacing for density reasons
> (e.g. `px-3.5`=14px, `gap-0.5`=2px, `gap-1.5`=6px). This phase does NOT clean up spacing —
> that refactor belongs to Phase 11. Declared below are the values in active use.

| Token | Value | Usage in public pages |
|-------|-------|-----------------------|
| xs | 4px (gap-1) | RealtimeDot gap |
| sm | 8px (gap-2) | Tab label + count gap |
| md | 16px (px-5, pt-4) | Page horizontal padding, section vertical padding |
| lg | 20px (py-5) | Card vertical padding (established; achieves ~68px tap target — passes ≥44px) |
| xl | 24px (p-8, pt-8) | Hero section padding, system-card padding |
| 2xl | 32px (py-8) | Footer area padding, list bottom padding |
| 3xl | 48px (py-20) | Empty state top/bottom padding |

Exceptions:
- `px-3.5` (14px) on tab buttons — inherited, do not change
- `gap-0.5` (2px), `gap-1.5` (6px), `gap-3.5` — Obsidian Blueprint density convention
- Non-multiple-of-4 values are intentional in this design system; cleanup deferred to Phase 11

> **Checker note (Dimension 5):** Spacing exceptions above are intentional — inherited
> from an established design system. Deferred to Phase 11 (UX-01). Do not flag as failure.

## Typography

> **Inherited system.** The existing pages use a wider type scale than the template default
> (necessity of the landing page marketing hierarchy). Documented verbatim from the
> actual components. Phase 11 owns any consolidation.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Micro label | 10–11px (`text-[10px]`, `text-[11px]`) | bold / normal | 1.0 (tracking-heavy) | Tab labels, card ref code, area_m2, price suffix, eyebrow labels |
| Small label | 12px (`text-[12px]`) | normal | 1.5 | Card building name, empty state body, back link |
| Body / Card heading | 22px (`text-[22px]`) | bold | tight (`leading-tight`) | Card unit name, card price value |
| Page heading | 32px (`text-[32px]`) | bold | none (`leading-none`) | `/unidades` page h1 |
| Landing display | `text-4xl..7xl` (responsive) | black / bold | tracked negative | Hero headline, section headings |

Weights in active use: normal (400), medium (500), semibold (600), bold (700), black (900).
Phase 9 introduces no new weights.

> **Checker note (Dimension 4):** 5 weights / 5 sizes exceed the "2 weights, 3-4 sizes" template
> default. This is intentional — the codebase uses a full marketing type scale established in
> Phase 4. Consolidation deferred to Phase 11. Do not flag as failure; the spec accurately
> reflects the files the executor works in.

#### D-02 — Hero CTA secondary: "VER PROJETOS" → "VER UNIDADES"

#### D-04 — SISTEMA.04 button: "ACESSE ANALITYCS" → "ACESSAR PAINEL"
- **Element:** Button in SISTEMA.04 system card (`src/app/page.js`)
- **Change:** Convert `<button type="button">` → `<Link href="/login">`
- **Label:** `ACESSAR PAINEL` (corrects typo "ANALITYCS"; label more precise)

#### D-05 — Card price fallback text
- **Element:** `src/components/features/UnidadePublicaCard.js`
- **Change:** `"Valor sob consulta"` → `"Consulte o Proprietário"` (when `valor_visivel = false`)
- **Style:** Preserve existing — `font-mono text-[11px] text-fg-3 tracking-[1px] uppercase`
- **Satisfies:** PUB-01

#### D-06 — Mobile tap targets: tab buttons ≥44px
- **Element:** Tab buttons in `src/components/features/UnidadesPublicas.js`
- **Current:** `py-2` → renders ~28-30px total height — fails ≥44px requirement
- **Change:** Update `py-2` → `py-3` AND add `min-h-[44px]`. Both are required: `py-3` honors the locked decision from CONTEXT.md D-06; `min-h-[44px]` guarantees the target since `py-3` alone (~38-40px) is insufficient.
- **Final className:** `px-3.5 py-3 min-h-[44px] inline-flex gap-2 font-body font-bold text-[10px] uppercase tracking-[0.5px] items-center border [...]`
- **Satisfies:** PUB-03

#### Additional tap target fix (required by PUB-03)
- **Element:** "← Voltar" back link in `src/components/features/UnidadesPublicas.js` line 82
- **Current:** No vertical padding → ~15px hit area — fails ≥44px
- **Change:** Add `py-3 inline-flex items-center min-h-[44px]` to the `<Link>` className
- **Satisfies:** PUB-03 (all tap targets ≥44px, including navigation links)

#### D-07 — Header "COMEÇAR AGORA" button (LP-03 compliance)
- **Element:** `src/components/ui/Header.js` — desktop nav (line 47) and mobile menu (line 96)
- **Current:** `<button type="button">` with no href — dead CTA that does nothing
- **Change:** Convert both instances to `<Link href="/login">` maintaining all existing styles
- **Label:** `COMEÇAR AGORA` (keep existing label — no copy change)
- **Style:** Preserve existing — `bg-linear-45 from-primary to-primary-hover py-4 px-10 ...`
- **Rationale:** LP-03 requires all buttons to have correct destinations. A dead button that silently does nothing is equivalent to a broken destination for a user who clicks it expecting to begin. The intent ("start now") maps cleanly to `/login`.
- **Satisfies:** LP-03

## Accessibility Notes

- All converted `<button>` → `<Link>` elements remain keyboard-navigable by default (Next.js `<Link>` renders as `<a>`)
- Tab buttons use `style={{ all: 'unset' }}` — ensure `cursor: pointer` is preserved in existing styles (it is, via `style` prop)
- Tap target minimum of 44px must be enforced via `min-h-[44px]` on both tab buttons and "← Voltar" link (PUB-03)
- `/unidades` in 375px viewport: verify no overflow-x after tab button height change; existing `overflow-x-auto [scrollbar-width:none]` on tab row handles width overflow
- Header nav links "CONTRATOS", "PORTAIS", "DASHBOARD" (href="#") are not `aria-disabled` — they are navigational placeholders that stay on the same page. These are acceptable for the banca demo scope; full nav wiring is post-TCC.
