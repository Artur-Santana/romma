## Typography

The existing dashboard uses a deliberate multi-scale system. Document the real scale with role assignments — do not collapse to 3–4 sizes, as the Console.OS aesthetic depends on scale contrast.

| Role | Size | Weight | Font | Line Height |
|------|------|--------|------|-------------|
| Micro label / mono caption | 9–11px | 700 | JetBrains Mono | 1.2 |
| Body / table data | 12–13px | 400–600 | Space Grotesk | 1.4 |
| Section heading | 24px | 700 | Space Grotesk (font-display) | 1.1 |
| Page heading mobile | 28px | 700 | Space Grotesk (font-display) | 1.1 |
| Sub-page heading | 32px | 700 | Space Grotesk (font-display) | 1.1 |
| Mobile metric display | 36px | 700 | Space Grotesk (font-display) | 1.0 |
| Desktop metric display | 48px | 700 | Space Grotesk (font-display) | 1.0 |

**Letter-spacing:** Display sizes use `letterSpacing: -2.4` at 48px, `letterSpacing: -1.8` at 36px, `letterSpacing: -1.5` at 32px, `letterSpacing: -1.4` at 28px, `letterSpacing: -0.5` at 24px.

**Eyebrow class:** `font-size: 10px`, `font-weight: 700`, `letter-spacing: 1.5px`, `text-transform: uppercase` — use `.eyebrow` utility class, never re-declare inline.

## States and Data Contract

### DASH-01: MRR Metric Tile

- **Location:** Dashboard main page, metric grid tile index 02 (desktop: 4-column grid; mobile: bottom-left of 2x2 grid)
- **Value formula:** `sum(unidades.valor_mensal WHERE contratos.status = 'ativo')`
- **Display format desktop:** 48px display number — if `mrr >= 1000`, show abbreviated (`R$12.5k`); else `fmtBRL(mrr)` — **check existing mobile pattern at line 371 of dashboard/page.js** which already abbreviates; desktop currently shows raw fmtBRL. Align both to the abbreviated pattern when mrr >= 1000.
- **Display format mobile:** Already implemented abbreviated format (`R$${(mrr/1000).toFixed(1)}k`).
- **Subline:** `"/ mês"` in mono 10px `--fg-4`
- **Empty state:** `"R$ 0,00"` or `"—"` when no active contracts (existing ghosted state uses `"—"`)

### DASH-02: Receita Esperada (to be added)

- **Location:** Dashboard main page — new metric replacing or augmenting "Parcelas Pendentes" tile (index 03). **Decision for planner:** REQUIREMENTS says "soma de parcelas com status pendente + vencida". Current code sums `valor_mensal` from the associated `unidade`, not the parcela itself. The UI label should read "Receita Esperada" (not "Parcelas Pendentes") and the value should display as `fmtBRL(receitaEsperada)` — a R$ currency amount.
- **Subline:** `"{N} parcela(s)"` count in mono 10px `--fg-4`
- **Value formula (note for planner):** Current implementation at `page.js:56-60` sums `unidade.valor_mensal` per pending parcela — this approximation may be acceptable for TCC scope. Planner to decide whether to sum actual parcela amounts or keep this. The UI contract is: display a R$ number labeled "Receita Esperada".

### DASH-03: Expiring Contracts Alert (already implemented)

- **Location:** Banner below metric grid. Appears only when `vencendoContratos.length > 0`.
- **Desktop:** Full-width banner, `border-left: 2px solid var(--warning)`, background `--warning-bg`, eyebrow `eyebrow--warning`, inline contract list, "Renovar →" link button.
- **Mobile:** Same banner pattern, margin `0 20px 24px`, shows first expiring contract + `+N` overflow indicator.
- **Metric tile 04 ("Vencendo em 7 dias"):** Already implemented with `warn: true` → `--warning-bg` background, `--warning` text color.
- **No changes required** to existing alert UI — it already meets DASH-03 requirements. Planner should verify this, not rebuild it.

### VIS-02: Visual Consistency Across Dashboard Screens

Target state for all four dashboard screens (Contratos, Parcelas, Unidades, Locatários):

**Pattern to match (reference: `dashboard/page.js`):**
- Page outer padding: `48px` on desktop
- Page header: `.eyebrow.eyebrow--indigo` label + `font-display` h2 at 48px weight 700 `letterSpacing: -2.4`
- Table container: `background: var(--surface)`, `border: 1px solid var(--border-3)`
- Table header row: `background: oklch(0.26 0 0)`, mono 10px `--fg-4` uppercase labels, `padding: 12px 20px`
- Table data rows: `padding: 16px 20px`, `borderTop: 1px solid var(--border-3)`
- Form inputs: `background: var(--surface-hi)`, `border: 1px solid var(--border-3)`, `color: var(--fg-1)`, `padding: 10px 14px`, `fontSize: 13`, `fontFamily: var(--font-mono)` — this is already the Contratos.js `inputStyle` pattern
- Action buttons (row-level, non-destructive): mono 10px `--fg-3` uppercase, no background border
- Destructive actions: use `ConfirmDialog` component — never inline delete without confirmation

**Known inconsistency (CLAUDE.md documented):** `Locatarios.js` uses Tailwind classes instead of inline styles. VIS-02 requires migration to inline + CSS vars to match the established canon. This is the main visual work for this phase.

## Layout Patterns

### Desktop (> 768px)

- Full-page wrapper: `.romma-desktop-only` + `.romma-page` (fade-in animation)
- Outer padding: `48px` horizontal and vertical (existing standard)
- Sidebar: `OwnerSidebar` via `DashboardLayout` — width managed by layout, not page
- Metric grid: `display: grid, gridTemplateColumns: repeat(4, 1fr)`, 1px solid `--border-3` outer border, inner column dividers
- Two-column content: `2fr 1fr` gap 32px (Contratos Recentes + Parcelas)
- Quick actions: 4-column equal grid

### Mobile (≤ 768px)

- `.romma-mobile-only` wrapper: `flex-direction: column, height: 100vh`
- `MobileTopBar` at top, `MobileBottomNav` at bottom
- Content scrollable `.romma-mobile-pane` (scrollbar hidden)
- Stats: 2x2 grid with `--border-3` borders
- Content list items: full-width, 20px margin from screen edge
