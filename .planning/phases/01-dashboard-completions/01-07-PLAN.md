<interfaces>
De src/lib/utils.js:
- cn(...classes) — merge de classes Tailwind

Mapeamento de tokens (mesmos do projeto):
- var(--success)   → text-success / bg-success
- var(--fg-1)      → text-fg-1
- var(--fg-2)      → text-fg-2
- var(--fg-4)      → text-fg-4
- var(--indigo)    → text-indigo / bg-indigo / border-indigo
- var(--font-mono) → font-mono
- var(--font-body) → font-body

Exceção técnica justificada para StatusBadge.js:
- StatusBadge usa `config.fg` e `config.bg` computados dinamicamente do STATUS_MAP.
- Valores são interpolados em runtime (ex: "oklch(from var(--success) l c h / 0.12)") — não mapeáveis para Tailwind.
- StatusBadge MANTÉM inline styles. Documentar no SUMMARY como exceção D-06 com justificativa técnica.

Exceção técnica para RealtimeDot animation:
- `animation: "rommaPulse 2s ease-in-out infinite"` — rommaPulse definido em globals.css @keyframes.
- Usar arbitrary value Tailwind: `[animation:rommaPulse_2s_ease-in-out_infinite]`.
</interfaces>
