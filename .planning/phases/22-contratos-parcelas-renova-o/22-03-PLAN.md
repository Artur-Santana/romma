<critical_facts>
- `authGuard()`, `UUID_RE`, `supabaseAdmin` já declarados no topo de contratos.js — REUSAR, não redeclarar.
- Cadeia de propriedade (3 níveis, igual cancelarContrato:85-98): contrato.unidade_id → unidades.edificio_id → edificios.eq('proprietario_id', user.id). Cada passo retorna 404 genérico ("Contrato não encontrado.") sem vazar qual registro falta.
- Validação de meses: `const m = Number(meses); if (!Number.isInteger(m) || m < 1 || m > 36) return { status: 400, erroMessage: 'Número de meses inválido (1–36).' }`. (Aceitar Number.isInteger para rejeitar NaN/decimais.)
- Datas SEMPRE com `+ 'T12:00:00'` ao construir Date (evita UTC shift / Pitfall 1, SC-5). `nova_data_fim`: `const d = new Date(contrato.data_fim + 'T12:00:00'); d.setMonth(d.getMonth() + m); const nova_data_fim = d.toISOString().slice(0,10)`.
- Numeração (Pitfall 5, D-16): NÃO usar `parcelas.length`. Buscar última parcela: `SELECT numero, data_fechamento FROM parcelas WHERE contrato_id=id ORDER BY numero DESC LIMIT 1`. `numero` da primeira nova = `lastParc.numero + 1`.
- Próximo fechamento: dia 1 do mês SEGUINTE ao último `data_fechamento` existente. `fechBase = new Date(lastParc.data_fechamento + 'T12:00:00'); fechBase.setMonth(fechBase.getMonth() + 1); fechBase.setDate(1)`.
- Loop `while (fechBase <= fim)` (fim = `new Date(nova_data_fim + 'T12:00:00')`): cada parcela `{ contrato_id: id, numero (incremental), data_fechamento: fechBase.toISOString().slice(0,10), data_vencimento: fechamento+7d, status: 'futura' }`. Vencimento: `const venc = new Date(fechBase); venc.setDate(venc.getDate()+7)`. Após push, `fechBase.setMonth(fechBase.getMonth()+1)`.
- Ordem: UPDATE data_fim PRIMEIRO, depois buscar última parcela, depois INSERT em lote. Se `novasParcelas.length === 0` (edge: renovação curta sem novo mês), ainda retornar `{ status: 200 }` (data_fim já atualizada).
- NUNCA invocar Edge Function `gerar-parcelas`. INSERT direto via `supabaseAdmin.from('parcelas').insert(novasParcelas)`.
- Retorno: `{ status: 200 }` ou `{ status: 4xx|5xx, erroMessage }`. Campo `erroMessage` (NÃO `errorMessage`).
- Modal (Parcelas.js): state form único `const [renew, setRenew] = useState({ meses: 0, custom: '' })` + `const [renovando, setRenovando] = useState(false)` + `const [showRenew, setShowRenew] = useState(false)` (Form State Contract).
- Botão "Renovar" vai no header de Parcelas, ao lado do StatusBadge (D-14). NÃO adicionar botão de renovação na tela de Contratos (CONTR-05 não pede).
- `--indigo` é usado SÓ no eyebrow "RENOVAÇÃO" do modal e no botão Confirmar (accent). Font weights 400 e 700.
- Plan 02 já redesenhou Parcelas.js — este plan ADICIONA o botão e o modal sobre essa base. Importar `renovarContrato` e `getContratos` (para re-fetch do contrato).
</critical_facts>
