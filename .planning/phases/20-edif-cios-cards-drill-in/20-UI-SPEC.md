## Layout Contract

### Page Layout

```
romma-page r-fade
  padding: var(--rd-page-y) var(--rd-gutter) + paddingBottom: 64px

  PageHeader                          (eyebrow + title + subtitle + CTA)
  [gap: var(--rd-block)]

  [showForm=true] CreateEdificioPanel (border-indigo, bg-surface, p-8, mb-8)

  [erro && !showForm] ErrorBanner

  CardGrid                            (grid 2-col on ≥ sm, 1-col on mobile)
    EdificioCard × N

  [modalState] UnifiedUnidadeModal    (romma-modal-backdrop overlay)
```

### Card Grid

```css
display: grid;
grid-template-columns: repeat(auto-fill, minmax(340px, 1fr));
gap: 16px;
```

- On viewport ≥ 720px: renders as 2 columns naturally (each card ~340px minimum).
- On viewport < 720px (mobile): collapses to 1 column.
- No explicit `grid-cols-2` class — `minmax(340px, 1fr)` achieves 2-col without hardcoding.

## Interaction Contract

### Accordion Expand / Collapse

- Toggle: click "Ver N unidade(s)" → `expandidos` Set state add/delete `edificio.id`.
- Expand state: `Set<string>` in a single `useState`. Survives data reload (separate from `edificios`/`unidades` state).
- Disabled: button has `disabled` attribute + `opacity: 0.4` + `cursor: not-allowed` when building has 0 units (D-04).
- Animation: conditional render (no JS height animation required). CSS `overflow: hidden` acceptable if animation desired, but not required by spec.
- Button label: `"Ver ${n} unidade${n !== 1 ? "s" : ""}"`

### Drill-in Unit Row Click

- Entire row is clickable (not just a button): `cursor: pointer`, `.r-rowlink` hover class.
- Opens `UnifiedUnidadeModal` with `mode="edit"`, `initial={unidade}`, `edificios={edificios}`, `lockEdificio={true}`, `onClose`, `onSaved`.
- `onSaved`: calls `carregarDados()` then `setModalState(null)`. Accordion stays open (expandidos not reset).
- Modal renders over `.romma-modal-backdrop` (existing utility class).

### lockEdificio in UnifiedUnidadeModal

- When `lockEdificio=true`: `FSelect` for "Edifício" receives `disabled={true}`. Options filtered to current `form.edificio_id` only (D-08). Visual cue: `opacity: 0.5`, `cursor: default` on the select element.
- `FSelect` extended: add `disabled` prop forwarded to inner `<select>` + opacity/cursor style.

### CRUD Actions (preserved — D-09)

| Action | Trigger | Behavior |
|--------|---------|----------|
| Criar edifício | PageHeader CTA → showForm toggle | Panel slides in above grid; form submitted via `criarEdificio` Server Action |
| Editar edifício | "Editar" ghost button on card header | Replaces card header row with InlineEditForm within the same card |
| Remover edifício | "Remover" danger button on card header | Calls `deletarEdificio` directly — no modal confirmation (existing behavior, not a new destructive flow per phase scope). If Server Action returns error (has units), error banner appears in page-level error zone. |

**Note:** "Remover" does NOT require a confirmation modal — the existing code calls `handleDeletar` directly. The Server Action blocks deletion if units exist (returns error). This phase preserves this existing behavior; no new confirmation dialog is introduced.

### Empty State (0 edifícios)

Rendered inside the card grid area — not a card, just a text block:

```
p className="r-body" style color fg-5 (or fg-4)
"Nenhum edifício cadastrado."
```

Consistent with existing `GestaoEdificios.js` line 168 empty state.

### Loading State (Skeleton)

`SkeletonEdificios` adapted for 2-column card grid:

```
grid grid-template-columns: repeat(auto-fill, minmax(340px, 1fr)), gap: 16px
  [0, 1, 2, 3].map → SkeletonCard
    border border-3, bg-surface, p-5
    Skeleton h-5 w-1/3 rounded-none   ← building name
    Skeleton h-3 w-2/3 mt-2           ← address
    Skeleton h-8 w-full mt-4          ← stats row
    Skeleton h-1.5 w-full mt-4        ← occupation bar
    Skeleton h-8 w-32 mt-4            ← expand button
```

## Stats Display Contract

Per-building stats derived client-side (D-01, D-02). Display format:

| Stat | Formula | Format | Color |
|------|---------|--------|-------|
| Nº de unidades | `unidades.length` | `N` (integer) | `--fg-1` |
| Ocupação % | `alugadas / total × 100` (0 if total=0) | `N%` (rounded, no decimals) | `--fg-1` |
| MRR | sum `valor_mensal` where `status === "alugada"` | `fmtBRLk(v)` → "R$ 12k" or "R$ 800" | `--highlight` (gold) |
| Área total | sum `area_m2` of all units in building | `N.NNN m²` (pt-BR locale, no compact) | `--fg-1` |

`fmtBRLk(v)` — redeclare locally in `GestaoEdificios.js` (copied from `Unidades.js` lines 62–71; not exported). Gold applies to MRR value + label to match Unidades.js "potencial em aberto" convention.

**Stat cell sub-label** (below the value, `.r-meta`):
- Nº unidades cell: `N alugada(s)` (count of alugadas)
- Ocupação cell: `N disponível(is)` (count of available)
- MRR cell: `N alugada(s)` (reinforces count)
- Área cell: `N unidade(s) · N alugada(s)` — or simply `N unidade(s)` (pick one; keep consistent)
