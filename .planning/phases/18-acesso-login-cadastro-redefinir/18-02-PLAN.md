    RED first — write test/unit/auth-form.test.js before src/lib/auth-form.js, run `npm run test:unit` and confirm it fails (module missing), commit the failing test, then implement.

    `maskPhone(value)`:
    - `maskPhone("11")` → `"(11"`; `maskPhone("1199998888")` (10 digits) → `"(11) 9999-8888"`; `maskPhone("11999998888")` (11 digits) → `"(11) 99999-8888"`.
    - Strips non-digits before masking; caps at 11 digits (`maskPhone("119999988887777")` → `"(11) 99999-8888"`).
    - No dangling trailing hyphen for partial input (`maskPhone("1199999")` has no trailing "-").

    `soDigitos(value)`:
    - `soDigitos("(11) 99999-8888")` → `"11999998888"`; strips all non-digits.

    `validarSenha(senha)` returns null when valid, else the exact policy message `"A senha deve ter no mínimo 6 caracteres, 1 letra maiúscula e 1 número."`:
    - `validarSenha("Ab1")` → message (too short); `validarSenha("abcdef1")` → message (no uppercase); `validarSenha("Abcdefg")` → message (no number); `validarSenha("Abc123")` → null (valid).

    `validarCadastro(form)` takes `{ nome, sobrenome, email, telefone, senha, confirmarSenha }`, returns null when all rules pass else the FIRST failing message (exact strings from UI-SPEC Validation Contract):
    - missing nome or sobrenome → `"Informe nome e sobrenome."`
    - invalid email (`/\S+@\S+\.\S+/` fails) → `"Informe um e-mail válido."`
    - telefone digits < 10 → `"Informe um telefone válido (com DDD)."`
    - weak senha → the validarSenha message
    - `senha !== confirmarSenha` → `"As senhas não coincidem."`
    - all valid (e.g. `{nome:"A", sobrenome:"B", email:"a@b.co", telefone:"(11) 99999-8888", senha:"Abc123", confirmarSenha:"Abc123"}`) → null
    - Rule order matters: assert phone is checked before password (a form failing both returns the phone message).
  </behavior>
