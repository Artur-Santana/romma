<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser form → Server Action (`cadastrarProprietario`) | Untrusted signup fields cross to server signUp |
| Browser → Supabase Auth (signInWithPassword) | Untrusted credentials cross to auth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-07 | Information Disclosure | login error banner | mitigate | Generic `Credenciais inválidas` message — no user-enumeration signal (same copy for unknown email vs wrong password). |
| T-18-08 | Spoofing | "manter sessão" implies stronger control than exists | accept | Cosmetic checkbox; session persistence is always-on via @supabase/ssr cookies (RESEARCH Pitfall 3). Honest because sessions do persist; no false security boundary created. |
| T-18-09 | Tampering | client validation bypassable | mitigate | UI gate via validarCadastro blocks normal path; Supabase enforces server-side password min + email format. soDigitos ensures only digits reach the DB (no injection surface — parameterized). |
| T-18-10 | Elevation of Privilege | supabaseAdmin import in client page | mitigate | signup page is a client component and imports ONLY the Server Action `cadastrarProprietario`, never supabaseAdmin. Enforced by `server-only` guard. |
| T-18-SC | Tampering | npm installs | mitigate | No package installs this phase (RESEARCH audit: none). |
</threat_model>
