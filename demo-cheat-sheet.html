Romma — Roteiro Demo Banca

18/06/2026  |  romma-alpha.vercel.app  |  Imprimir este arquivo (sem credenciais)

Parte 1 — Listagem Pública ~3 min
1.Abrir /unidades em janela anônima
2.Mostrar cards: nome, área, valor mensal
3.Apontar card "Consulte o Proprietário" (valor oculto)
4.Notar: unidades alugadas NÃO aparecem

Destaque

Server Component — RLS filtra disponível no servidor, sem JS no cliente.

Parte 2 — Dashboard Proprietário ~4 min
5.Logar como Proprietário → /dashboard
6.Apontar KPIs: MRR e Receita Esperada
7.Mostrar alerta DASH-03 (parcela vencida)
8.Contratos → clicar contrato → ver parcelas
9.Mostrar status: futura / pendente / paga / vencida

Destaque

Status calculado por data — sem cronjob. Determinístico e auditável.

Parte 3 — Edge Function Parcelas ~3 min
10.Criar contrato para unidade disponível
11.Clicar "Gerar Parcelas"
12.Mostrar parcelas criadas com datas calculadas

Destaque

Edge Function Deno — transação atômica, autenticada por JWT próprio (não anon key).

Parte 4 — Realtime ~2 min
13.Abrir /unidades em 2ª aba (sem login)
14.No dashboard: criar contrato para unidade visível
15.Voltar para 2ª aba — card deve sumir

Fallback — se card NÃO sumir:

Pressionar F5 e explicar: "RLS do Supabase filtra eventos UPDATE de disponível→alugada — limitação conhecida, workaround é refresh ou polling."

Parte 5 — Portal Locatário ~3 min
16.Nova janela anônima → /locatario
17.Login com conta demo pré-configurada
18.Mostrar contrato ativo e parcelas

Destaque

RLS isola locatário — vê apenas seus dados via auth.uid(). Zero acesso ao dashboard.

Parte 6 — Convite (opcional) ~2 min
19.Dashboard → Locatários → Convidar
20.Inserir email → mostrar confirmação

Destaque

Admin API Supabase — sem senha temporária exposta, redirect direto para /dashboard em prod.
